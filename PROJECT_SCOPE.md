# Project Scope: Human-AI Decision Record Tool

## Distinction from the AI Capability Framework
The **AI Capability Framework** is designed to develop human capability—it provides a map of the skills, knowledge, and aptitudes required to interact with AI responsibly. It is an educational and developmental structure.

In contrast, the **Human-AI Decision Record Tool** does not aim to assess or teach capability. Instead, it assumes capability needs to be *exercised* and *governed*. It is a discrete software instrument used at the point of action.

## Relation to Capability-Driven Development (CDD)
This application is a live demonstration of **Capability-Driven Development (CDD)**. 
CDD is a design methodology for building systems around capability and governance requirements. This project proves that governance needs (traceability, accountability, reviewability) can fundamentally shape the architecture and user experience of a software tool from day one, rather than being added as afterthoughts or compliance checks.

## Positioning within Human-AI Governance Engineering
This tool belongs under the **Human-AI Governance Engineering** domain. This domain applies engineering practices to design governance-ready human-AI systems. The Human-AI Decision Record Tool is a concrete software application implementing these principles, prioritizing inspectability over abstraction, and giving professionals practical means to operationalize human accountability when working with AI.
